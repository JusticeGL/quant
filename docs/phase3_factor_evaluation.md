# Phase 3：自定义因子固定评价

## 边界

Phase 3 建立因子实现和元数据契约、候选注册表、固定结构化评价以及静态/动态防泄漏
门禁。它提供三个手工参考因子：20 日动量、5 日反转和 20 日实现波动率。

本阶段不会自动生成新因子，不会自动接受或拒绝候选，也不会运行锁定 test。所有
`eligible_for_review` 只表示通过配置化工程门槛，可供人工复核。

## 因子契约

每个 ID 对应一个 Python 文件和一个 YAML 文件。YAML 声明名称、假设、公式、输入、
lookback、预期方向、家族、作者、父因子和状态。运行时只把主键和声明输入传入函数，
并验证：

- 输入未被修改；
- 输出列严格为 `trade_date/instrument/value`；
- 输出键来自输入且没有重复；
- value 为数值、允许 NaN、禁止正负无穷；
- 实现源码和元数据分别记录 SHA256。

统一截面处理位于锁定评价代码中：按日分位数去极值、按方向调整并做 z-score。候选
不得自行改变评价口径。

## 防泄漏

静态 AST 检查禁止负向 `shift/pct_change`、居中 rolling、缺失 `min_periods`、标签字段、
未声明输入、网络库和文件写入。动态检查要求：

1. 使用完整输入和截至某日的截断输入时，该日及之前结果完全一致；
2. 修改该日之后的所有数值输入时，历史结果完全一致。

`tests/fixtures/factors/F9999.py` 故意使用 `shift(-1)`，完整测试必须确认它被拒绝。

## 固定评价输出

评价只使用 validation，标签结果仅允许落在锁定 test 之前的隔离日。输出包括：

- 覆盖率、Pearson IC、RankIC、标准差、ICIR 和正 IC 比例；
- 月度、年度和三个滚动子区间的方向稳定性；
- 五分组收益、单调性和 Top-minus-Bottom；
- 横截面排名换手率；
- 与所有注册因子的相关性，以及与已接受因子的最大相关性；
- 缺失率、无穷值和分位数分布；
- Top-K 的零成本、基础成本、双倍成本结果和成本符号稳定性；
- 数据快照、配置哈希、源码哈希、Git commit 和 test 未访问声明。

历史行业和市值字段尚未进入 silver，因此暴露结果明确为 `unavailable`。牛熊阶段政策
也尚未定义，报告不会自行猜测市场状态。

## 注册与审计

成功评价会登记到 DuckDB 的 `research.factor_definition`、`factor_version`、
`experiment_run`、`experiment_metric` 和 `backtest_run`，大体量因子值仍保存在被 Git
忽略的 Parquet 中。跨容器目录写入使用文件锁串行化，以符合 DuckDB 单写者模型；因子
计算仍可并行。失败的评价结果和测试证据不得因结果不好而删除。

当前样本仅为 10 只股票、2024 年上半年，存在生存者偏差，所有输出仅用于工程验证。
